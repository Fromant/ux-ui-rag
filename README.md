# RAG-система для учебника по дискретной математике

Локальная система поиска и тренажёр для учебника по дискретной математике (DM2024.pdf). **Работает полностью офлайн** - не требует LLM или API.

**Важно:** Учебник и сгенерированные PNG-файлы **не** хранятся в репозитории. При первом запуске нужно положить PDF в `books/DM2024.pdf` и запустить скрипт генерации данных.

## Возможности

- **Поиск по учебнику** - BM25 полнотекстовый поиск по секциям с ключевыми словами
- **Тренажёр/Quiz** - режим проверки знаний с разнообразными вопросами (шаблонная генерация)
- **Умные ключевые слова** - извлечение из красного текста + предметного указателя
- **Просмотр страниц** - PNG-изображения страниц учебника с навигацией
- **Навигация** - загрузка дополнительных страниц в обе стороны

## Требования

- Docker и Docker Compose
- Python 3.11+ (только для генерации индекса)
- CPU-only
- ~2.5GB RAM
- ~500MB disk space
- **Не требует API ключей** - всё работает офлайн!

## Быстрый старт

### 1. Подготовка данных (один раз)

```bash
# Положить учебник в books/DM2024.pdf

# Создать venv и установить зависимости для генерации
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt

# Запустить генерацию индекса с ключевыми словами и вопросами
python build_keywords.py --pdf books/DM2024.pdf --output data/sections_index.json

# Или без генерации вопросов (быстрее):
python build_keywords.py --pdf books/DM2024.pdf --skip-questions
```

Генерация занимает 2-5 минут (зависит от размера PDF). Данные сохранятся в `data/`.

### 2. Запустить приложение

```bash
docker compose up --build -d
```

### 3. Открыть в браузере

`http://localhost:8000`

## Доступ к сгенерированным данным

Все данные доступны в локальной папке `./data`:

- `data/pages/` - PNG изображения страниц учебника
- `data/sections_index.json` - индекс секций с вопросами для тренажера

## Архитектура

### Backend (FastAPI)

**main.py** - основное приложение:
- `POST /api/search` - поиск по учебнику
- `GET /api/pages/{page_num}` - получить диапазон страниц
- `POST /api/simulator/next` - следующий вопрос (с типом и сложностью)
- `POST /api/simulator/validate` - проверить ответ
- `GET /data/pages/{filename}` - отдать PNG страницу

**build_keywords.py** - pipeline генерации индекса:
- `SectionExtractor` - извлечение секций из PDF по паттернам
- `RedTextKeywordExtractor` - определение красного текста (ключевые слова)
- `SubjectIndexParser` - парсинг предметного указателя
- `QuestionTemplateGenerator` - шаблонная генерация вопросов

**app/processors/pdf_keyword_extractor.py** - умное извлечение ключевых слов:
- **Красный текст**: Использует `fitz.Page.get_text("dict")` для получения цветовых данных
  - Определяет RGB/CMYK цвета спанов
  - Фильтрует текст с красным цветом (R >= 0.8, G,B <= 0.3)
  - Возвращает список ключевых слов по страницам
- **Предметный указатель**: Парсинг последних 50 страниц PDF
  - Ищет заголовок "Предметный указатель" или аналогичные
  - Извлекает пары "ключевое слово → номера страниц"
- **Слияние источников**: Приоритет: предметный указатель > красный текст > частотный анализ

**app/processors/question_template_generator.py** - шаблонная генерация вопросов:
- Генерирует 3 разнообразных вопроса на секцию используя шаблоны:
  - Вопросы-определения ("Что такое...", "Дайте определение...")
  - Вопросы на понимание ("Объясните...", "Расскажите о...")
  - Специфические вопросы (теоремы, алгоритмы, свойства)
  - Вопросы на основе ключевых слов ("Как связано X с Y?")
- Не требует LLM - работает полностью офлайн
- Вопросы генерируются на этапе сборки и сохраняются в индексе

**app/search/simple_search.py** - BM25 поиск:
- Использует библиотеку `rank-bm25`
- Индекс строится из заголовков секций и ключевых слов
- Возвращает top-k результатов с оценками релевантности

**app/models/answer_validator.py** - проверка ответов:
- Использует sentence transformers для семантического анализа
- Модель `paraphrase-multilingual-MiniLM-L12-v2` (поддерживает русский)
- Сравнивает эмбеддинги ответа студента с текстом секции
- Возвращает оценку схожести и уровень уверенности

### Frontend

**templates/index.html** - страница поиска:
- Вертикальный скролл результатов
- Кнопки "Предыдущие 5" / "Следующие 5" для загрузки страниц учебника
- Показ диапазона загруженных страниц
- Click-to-expand для просмотра страниц

**templates/simulator.html** - тренажёр:
- Вопрос показывается БЕЗ контекста
- После ответа студента показываются страницы учебника
- Оценка ответа через семантическую схожесть (sentence transformers)
- Разные уровни уверенности: high/medium/low/very_low
- Развёрнутая обратная связь в зависимости от уверенности


## Как это работает

### Умное извлечение ключевых слов

Система использует три источника ключевых слов с приоритетами:

1. **Предметный указатель** (высший приоритет)
   - Парсится из последних 50 страниц PDF
   - Формат: "Ключевое слово, 123, 145, 150"
   - Самый надёжный источник - авторские ключевые слова

2. **Красный текст** (средний приоритет)
   - Определяется через PyMuPDF text dict с цветовой информацией
   - RGB: R >= 0.8, G <= 0.3, B <= 0.3
   - Ключевые слова часто выделены красным в учебнике

3. **Частотный анализ** (резервный)
   - Стандартный regex-based подсчёт слов
   - Фильтрация стоп-слов и кодовых артефактов
   - Используется если другие источники не нашли ключевых слов

### Генерация вопросов

Шаблоновая система генерирует 3 вопроса на секцию:
- **Определения**: "Дайте определение: ..."
- **Понимание**: "Объясните, что такое ..."
- **Специфические**: зависит от типа секции (теорема, алгоритм, свойство)
- **Ключевые слова**: "Как связано X с Y?"

Все вопросы генерируются на этапе сборки - не требует LLM или API!

### Поиск

1. Пользователь вводит запрос (например, "графы")
2. BM25 вычисляет релевантность для каждой секции
3. Возвращаются top-k результатов (по умолчанию 3)
4. Для каждого результата показывается главная страница +-2 (всего 5)
5. Frontend позволяет листать вверх/вниз кнопками +-5

### Индексация

1. PDF открывается через PyMuPDF
2. Регуляркой ищутся заголовки вида `1.2.3. Название`
3. Заголовки очищаются от артефактов (2/2), обрывков, случайных имён
4. **Извлекаются красные ключевые слова** (цветовой анализ)
5. **Парсится предметный указатель** (авторитетный источник)
6. **Генерируются шаблонные вопросы** (3 на секцию)
7. Ключевые слова сливаются с приоритетом
8. Результат сохраняется в JSON

## Размеры

- Исходный PDF: ~30MB
- PNG изображения: ~167MB (1738 страниц)
- Индекс секций: ~150KB (~400 чистых записей)
- Модель sentence-transformers: ~120MB (кэшируется)
- Docker образ: ~1.2GB (включает torch и модель)

## Ограничения

- Семантическая валидация работает хорошо, но не идеальна (порог 0.35 cosine similarity)
- Модель требует ~500MB RAM и добавляет ~100-200ms на проверку ответа
- Все секции индексируются, даже оглавление/приложения
- **Не требует API** - всё работает офлайн!
- Качество извлечения красных ключевых слов зависит от форматирования PDF
- Предметный указатель может отсутствовать в некоторых учебниках
